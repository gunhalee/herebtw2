# 여기 근데 한마디 할게요 — Product Requirements Document

## 1. 개요

### 1-1. 제품 요약

"여기 근데 한마디 할게요"는 지방선거 기초지방자치단체 의원 후보 출마자와 지역 주민 간의 위치 기반 익명 소통 플랫폼이다. 유권자는 계정 생성 없이 자신의 위치와 함께 익명으로 의견을 남기고, 관리자에 의해 인증된 후보 출마자만이 해당 의견에 답변할 수 있다.

### 1-2. 문제 정의

지방선거에서 기초의원 후보는 유권자와 직접 소통할 채널이 부족하다. 기존 SNS는 팔로워 기반이라 도달 범위가 제한적이고, 유세 현장은 일방적 전달에 그친다. 유권자 역시 자신의 의견이 후보에게 전달되는 경로가 없으며, 후보가 어떤 약속을 했는지 추적할 수단이 없다.

### 1-3. 제품 목표

- 유권자가 자기 동네에서 가장 쉬운 방법으로 목소리를 낼 수 있는 채널을 만든다.
- 후보가 유권자의 의제에 응답하는 구조를 만들어, 소통의 주도권을 유권자에게 둔다.
- 소통 기록이 남아 후보의 약속에 대한 책임을 구조적으로 부여한다.
- 선거 이후에도 지속 가능한 소통 아카이브를 만든다.

### 1-4. 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| 유권자 경험은 단순하게 | 유권자에게는 새로운 학습을 요구하지 않는다. 복잡한 기능은 후보 쪽에만 얹는다. |
| 유권자가 의제의 주인 | 좋아요 기반 정렬로 어떤 목소리가 중요한지 유권자 커뮤니티가 결정한다. 후보는 이 순서를 바꿀 수 없다. |
| 모든 기능은 기록으로 수렴 | 글, 답변, 약속은 삭제·수정되지 않는다. 기록이 남는다는 사실 자체가 책임을 만든다. |
| 비대칭 소통 구조 | 유권자의 익명 글은 가볍게, 후보의 실명 답변은 무겁게. 이 비대칭이 서비스의 철학이다. |

## 2. 사용자 정의

### 2-1. 유권자 (비인증 사용자)

- 계정이 없다. 로그인하지 않는다.
- 위치 권한 허용만으로 글을 남길 수 있다.
- 모든 글을 읽을 수 있고, 좋아요를 누를 수 있다.
- 답글을 달 수는 없다.

### 2-2. 후보 출마자 (인증 사용자)

- 관리자에 의해 인증된 후보.
- 자기 지역구 글에 답변을 달 수 있다.
- 답변 시 "약속합니다" 태그를 붙일 수 있다.
- 전용 대시보드를 통해 미답변 현황을 관리한다.

### 2-3. 관리자

- 후보 계정을 생성·관리한다.
- 부적절한 글을 관리한다.
- 서비스 설정(선거일, 지역구 등)을 관리한다.

## 3. 현재 상태 (v0)

현재 구현되어 있는 기능:

- 위치 권한 허용 기반 자동 위치 입력
- 익명 글 작성 (위치 + 본문)
- 글 목록 표시 (위치, 거리, 작성 시간)
- 좋아요 기능

## 4. 기능 요구사항

### Phase 1 — 유권자 경험 핵심 루프

유권자가 "내 글을 다시 찾고, 답변을 기다릴 수 있는" 연결고리를 만든다.

#### F1-1. 고유 링크 생성 및 제공

**설명**: 글 작성 완료 시 해당 글에 접근할 수 있는 고유 URL을 생성하고, 유권자에게 제공한다.

**요구사항**:
- 글 작성 완료 시 UUID 기반 고유 링크를 자동 생성한다.
- URL 형태: `herebtw.vercel.app/v/{uuid}`
- 완료 화면에 아래 요소를 표시한다:
  - 체크 아이콘 + "당신의 목소리가 전달되었습니다" 메시지
  - 링크 복사 버튼
  - 복사 완료 시 "복사됨" 피드백 (토스트 또는 버튼 텍스트 전환)
- UUID는 글의 내부 PK와 별도로, 외부 노출 전용 식별자로 사용한다.

**데이터 변경**:
- `posts` 테이블에 `uuid` 필드 추가 (UNIQUE, NOT NULL, 자동 생성)

#### F1-2. 고유 링크 페이지

**설명**: 유권자가 고유 링크를 통해 자신의 글과 현재 상태를 확인할 수 있는 페이지.

**요구사항**:
- 표시 정보: 글 본문, 위치, 작성 시간
- 상태 표시 (2단계):
  - **전달됨**: 글 작성 직후 기본 상태. 아이콘 + "전달됨" 텍스트.
  - **답변 완료**: 후보 답변이 등록된 상태. 아이콘 + "답변이 도착했습니다" 텍스트.
- 답변 완료 상태일 경우, 글 아래에 후보의 답변을 표시한다:
  - 후보명 + 인증 배지
  - 답변 본문
  - "약속합니다" 배지 (해당 시)
  - 답변 작성 시간
- 포토카드 다운로드 버튼 (Phase 3 연동, Phase 1에서는 미노출)

**데이터 변경**:
- `posts` 테이블에 `status` 필드 추가: enum (`delivered`, `replied`)
- 기본값: `delivered`
- 답변 등록 시 자동으로 `replied`로 전환

#### F1-3. 이메일 알림

**설명**: 유권자가 선택적으로 이메일을 남기면, 후보 답변 시 알림을 받을 수 있다.

**요구사항**:
- 글 작성 화면에 이메일 입력 필드를 추가한다.
  - 필수가 아닌 선택 사항.
  - 안내 문구: "답변이 달리면 알려드릴까요?"
  - 이메일 형식 유효성 검사.
- 후보가 답변을 등록할 때, 해당 글에 이메일이 등록되어 있으면 알림을 발송한다.
- 알림 이메일 내용:
  - 제목: "당신의 목소리에 답변이 달렸습니다"
  - 본문: 원본 글 요약 + 고유 링크 버튼
  - 발신자: 서비스명
- 이메일은 해당 글의 답변 알림 용도로만 사용하며, 마케팅 등 다른 목적으로 활용하지 않는다.
- 개인정보 수집 고지 문구를 이메일 입력 필드 하단에 표시한다.

**데이터 변경**:
- `posts` 테이블에 `notification_email` 필드 추가 (varchar, nullable)

**외부 서비스**:
- 이메일 발송: Resend 또는 동급 트랜잭셔널 이메일 서비스

### Phase 2 — 후보 인증 및 답변 시스템

후보가 인증하고, 유권자의 목소리에 답변하고, 공약을 기록할 수 있는 구조를 만든다.

#### F2-1. 후보 인증 플로우

**설명**: 관리자가 생성한 계정으로 후보가 로그인하고, 서비스에 공적으로 진입하는 과정.

**요구사항**:
- 관리자가 후보 계정을 생성한다 (이름, 지역구, 이메일).
  - 초대 코드 또는 이메일 기반 초대 링크 발송.
- 후보는 초대를 수락하고 비밀번호를 설정하여 계정을 활성화한다.
- 최초 로그인 시 "첫 마디" 작성을 필수로 요구한다.
  - 첫 마디 미작성 시 다른 기능 접근 불가.
- 첫 마디 표시 규칙:
  - 일반 주민 글과 동일한 카드 형태로 표시한다.
  - 후보명 + 인증 배지를 카드 상단에 표시한다.
  - 해당 지역구 글 목록에서 최상단에 고정한다.
  - 좋아요 기능은 일반 글과 동일하게 적용한다.

**데이터 변경**:
- `candidates` 테이블 신규 생성
- `posts` 테이블에 `is_pinned` (boolean, default false), `author_type` (enum: `citizen`, `candidate`), `candidate_id` (FK, nullable) 필드 추가

#### F2-2. 후보 대시보드

**설명**: 후보가 자기 지역구의 유권자 목소리를 관리하고 답변하는 전용 화면.

**요구사항**:
- 후보 로그인 후 접근 가능한 전용 페이지.
- 자기 지역구의 글만 표시한다.
- 기본 정렬: 미답변 글 우선 → 좋아요순.
- 각 글 카드에 표시할 정보:
  - 글 본문, 위치, 작성 시간, 좋아요 수
  - 답변 여부 아이콘 (미답변: 빈 말풍선 / 답변 완료: 채워진 말풍선)
- 미답변 + 좋아요 N개 이상 글 강조:
  - 카드 배경색 변경 또는 "주민 N명이 관심을 보인 목소리입니다" 배지 표시.
  - N의 기본 임계값: 3 (관리자 설정 가능).
- 상단 통계 요약:
  - 총 글 수 / 답변 완료 수 / 미답변 수 / 답변률(%)
- 글 카드 클릭 시 답변 작성 화면으로 진입.

#### F2-3. 답변 작성

**설명**: 후보가 유권자의 글에 답변을 작성하는 기능. 답변은 공적 기록으로 남으며 수정할 수 없다.

**요구사항**:
- 답변 작성 진입: 후보 대시보드의 글 카드 또는 일반 글 목록에서 접근.
- 답변 본문 입력:
  - 최대 200자.
  - 입력 중 잔여 글자 수 실시간 표시.
  - 200자 초과 시 입력 차단.
- 수정 불가 정책:
  - "등록" 버튼 클릭 시 확인 모달 표시: "답변은 등록 후 수정할 수 없습니다. 등록하시겠습니까?"
  - 확인 후 등록, 취소 시 작성 화면으로 복귀.
- 등록 시 처리:
  - 답변 데이터 저장.
  - 해당 글의 상태를 `replied`로 전환.
  - 글에 등록된 이메일이 있으면 알림 발송 트리거.
- 답변 표시 (글 목록 및 고유 링크 페이지):
  - 원본 글 카드 하단에 답변 영역 표시.
  - 후보명 + 인증 배지 + 답변 본문 + 작성 시간.
  - "약속합니다" 배지 (해당 시).
- 한 글에 대해 하나의 답변만 허용한다.

**데이터 변경**:
- `replies` 테이블 신규 생성

#### F2-4. "약속합니다" 기능

**설명**: 후보가 답변에 공약 성격의 약속을 명시적으로 표시하고, 기한을 설정하는 기능.

**요구사항**:
- 답변 작성 화면에 "이 답변은 약속입니다" 토글을 추가한다.
- 토글 활성화 시 기한 선택 필드가 나타난다:
  - 드롭다운 옵션: "당선 후 3개월" / "당선 후 6개월" / "당선 후 1년" / "직접 입력"
  - "직접 입력" 선택 시 날짜 선택 UI.
- 약속이 포함된 답변의 표시:
  - 답변 카드에 "약속합니다" 배지 + 기한 텍스트.
  - 예: `[약속합니다] 당선 후 6개월 이내`
- 후보 프로필 페이지에 약속 목록 자동 축적 (Phase 2-4와 F3-3 연동).

**데이터 변경**:
- `replies` 테이블의 `is_promise` (boolean, default false), `promise_deadline` (date, nullable) 필드

### Phase 3 — 정렬, 포토카드, 공약 아카이브

유권자 주도권을 구조적으로 보장하고, 양방향 바이럴 장치를 만들고, 선거 이후 연속성을 확보한다.

#### F3-1. 거리 기반 정렬 + 동네 내 좋아요순

**설명**: 유권자에게 자기 동네의 중요한 목소리를 먼저 보여주는 정렬 체계.

**요구사항**:
- 1단계 정렬 — 거리 기반 동네 그룹핑:
  - 접속 사용자의 현재 위치 기준, 가까운 동네(행정동 단위)의 글부터 표시한다.
  - 동네 구분 기준: 글의 `location_name` 필드 (행정동 단위).
  - 동일 동네 글은 하나의 그룹으로 묶어 표시한다.
- 2단계 정렬 — 동네 내 정렬:
  - 기본: 좋아요순 (시간 감쇠 적용).
  - 전환 가능: 최신순.
  - 정렬 모드 전환 UI: "공감순 / 최신순" 토글 (글 목록 상단).
- 시간 감쇠:
  - 좋아요 스코어에 시간 감쇠를 적용하여, 오래된 글의 좋아요가 최신 글보다 과도하게 우세하지 않도록 한다.
  - 감쇠 함수 예시: `score = likes / (1 + hours_since_post / 72)` (72시간 = 3일 반감기)
  - 감쇠 파라미터는 관리자 설정으로 조정 가능하게 한다.
- 초기 대응:
  - 동네 내 글 수가 임계치(기본 10개) 미만일 경우, 최신순을 기본 정렬로 적용한다.
  - 임계치 초과 시 좋아요순으로 자동 전환.
- 미답변 강조:
  - 좋아요가 임계값(기본 3) 이상이면서 미답변인 글에 "주민 N명이 관심을 보인 목소리입니다" 배지를 표시한다.
  - 이 배지는 후보 대시보드뿐 아니라 일반 글 목록에서도 표시한다.
- 후보 첫 마디(pinned) 글은 정렬과 무관하게 해당 지역구 최상단에 고정한다.

#### F3-2. 포토카드

**설명**: 유권자와 후보 모두가 소통 기록을 이미지로 만들어 SNS에 공유할 수 있는 기능.

**요구사항**:

*카드 종류 및 비주얼*:
- 상태별 2종:
  - **전달됨 카드**: 유권자 글만 표시. 차분한 기본 톤.
  - **답변 완료 카드**: 유권자 글 + 후보 답변. 대화가 완성된 느낌. "약속합니다" 배지 포함 (해당 시).
- 시점별 2종:
  - **유권자 카드**: "나는 목소리를 냈다"는 관점. 유권자의 글이 주인공.
  - **후보 카드**: "나는 주민의 목소리에 답했다"는 관점. 후보의 답변이 주인공.
- 공통 요소:
  - 위치 (행정동명)
  - 누적 번호: "OO동에서 N번째 목소리" (해당 동네에서의 글 순번)
  - 서비스 로고 + URL
  - 서비스 QR 코드 (선택적 — 카드 크기에 따라 포함 여부 결정)

*다운로드 진입점*:
- 글 작성 직후 완료 화면: "전달됨" 카드 다운로드 버튼.
- 고유 링크 페이지: 현재 상태에 맞는 카드 다운로드 버튼.
- 후보 대시보드: 답변 완료 글에서 후보용 카드 다운로드 버튼.

*이미지 생성*:
- 서버사이드 이미지 생성: API 엔드포인트 `GET /api/card/{uuid}?type=voter|candidate`
- PNG 형식, 해상도: 2x (SNS 공유 최적화).
- 카드 크기: 1080×1350px (인스타그램 4:5 비율) 또는 1200×630px (트위터/페이스북 OG 비율). 주 사용 SNS에 따라 결정.

*OG 메타태그*:
- 고유 링크(`/v/{uuid}`) 공유 시, 해당 카드 이미지를 `og:image`로 동적 생성하여 SNS 미리보기에 표시한다.
- `og:title`: "OO동에서 N번째 목소리"
- `og:description`: 글 본문 앞 50자

**외부 서비스/라이브러리**:
- 서버사이드 이미지 생성: satori + @resvg/resvg-js (경량) 또는 Puppeteer (고품질)

#### F3-3. 공약 아카이브 페이지

**설명**: 후보별 약속을 모아보는 페이지. 선거 이후에도 약속의 기록과 기한을 확인할 수 있다.

**요구사항**:
- URL: `/candidate/{id}/promises`
- 후보 정보 표시:
  - 후보명, 지역구
  - 요약 통계: "주민 N명에게 답하고 M개의 약속을 한 후보"
  - 답변률: 전체 글 대비 답변 비율
- 약속 목록:
  - 각 약속마다 표시:
    - 원본 주민 글 (본문, 위치, 작성 시간)
    - 후보 답변 (본문, 작성 시간)
    - 약속 기한
    - D+일수: 선거일(관리자 설정값) 이후 경과일 자동 표시
  - 정렬: 기한 임박순 기본
- D+일수 표시 규칙:
  - 선거일 이전: "선거일까지 D-N"
  - 선거일 이후 ~ 기한 이전: "D+N / 기한까지 M일"
  - 기한 경과: "기한 D+N 경과"
- 이 페이지는 비인증 사용자(유권자)도 접근 가능하다.
- 선거 이후에도 페이지를 유지하여, 서비스의 연속성 장치로 활용한다.

**데이터 변경**:
- 관리자 설정 테이블에 `election_date` (date) 필드 추가

## 6. 사용자 흐름

### 6-1. 유권자 — 글 작성 및 추적

```
서비스 접속
  → 위치 권한 허용 (자동 위치 입력)
  → 글 작성 (본문 입력 + 이메일 입력(선택))
  → 작성 완료 화면
      - "당신의 목소리가 전달되었습니다"
      - 고유 링크 복사 버튼
      - 포토카드 다운로드 버튼 (Phase 3)
  → (이후) 고유 링크로 재방문
      - 상태 확인: 전달됨 → 답변 완료
      - 답변 내용 확인
      - 업데이트된 포토카드 다운로드 (Phase 3)
  → (답변 등록 시) 이메일 알림 수신
```

### 6-2. 후보 — 인증 및 답변

```
관리자로부터 초대 수신
  → 계정 활성화 (비밀번호 설정)
  → 최초 로그인
      - 첫 마디 작성 (필수)
      - 작성 완료 → 글 목록 최상단 고정
  → 대시보드 접근
      - 미답변 글 확인 (좋아요순, 강조 표시)
      - 글 선택 → 답변 작성
          - 본문 입력 (200자 제한)
          - "약속합니다" 토글 (선택) + 기한 설정
          - 확인 모달 → 등록
      - 답변 완료 → 포토카드 다운로드 (Phase 3)
  → 프로필 페이지 / 공약 아카이브 자동 축적
```

### 6-3. 유권자 — 글 탐색

```
서비스 접속
  → 글 목록 표시
      - 가까운 동네 글부터 (거리 기반)
      - 동네 내 좋아요순 (기본) / 최신순 전환
      - 후보 첫 마디: 해당 지역구 최상단 고정
      - 미답변 강조 배지 표시
  → 좋아요 누르기
  → 후보 답변 확인
  → 후보 프로필 / 공약 아카이브 열람
```

## 7. 비기능 요구사항

### 7-1. 개인정보 보호

- 유권자의 위치 데이터는 행정동 단위까지만 저장·표시한다. 정확한 좌표는 거리 계산용으로만 사용하고, 목록에서는 행정동명만 노출한다.
- 이메일은 해당 글의 답변 알림 용도로만 사용하며, 마케팅·분석·제3자 제공 등에 활용하지 않는다.
- 이메일 수집 시 개인정보 수집·이용 동의 고지를 표시한다.

### 7-2. 데이터 무결성

- 후보 답변은 등록 후 수정·삭제할 수 없다 (관리자 예외).
- "약속합니다" 태그 및 기한은 등록 후 변경할 수 없다.
- 유권자 글 삭제 정책: 유권자는 글을 삭제할 수 없다 (익명이므로 본인 확인 불가). 관리자만 부적절한 글을 비공개 처리할 수 있다.

### 7-3. 성능

- 글 목록 로딩: 첫 화면 1초 이내.
- 거리 기반 정렬: 접속자 위치 확보 후 500ms 이내 정렬 완료.
- 포토카드 이미지 생성: 3초 이내.
- 이메일 알림 발송: 답변 등록 후 1분 이내.

### 7-4. 접근성

- 모바일 퍼스트 반응형 디자인. 주 사용 환경은 모바일 브라우저.
- 글 작성에 계정·로그인이 필요 없어야 한다.

## 8. 개발 일정 및 우선순위

### 의존 관계

```
Phase 1 ───→ Phase 2 ───→ Phase 3
(고유 링크)    (답변 시스템)   (포토카드·아카이브)
                  ↓
           이메일 알림 연동
                  ↓
            공약 아카이브 연동
```

- Phase 1 완료 → Phase 2 착수: 고유 링크 페이지가 있어야 답변 상태 전환과 이메일 알림이 의미를 가진다.
- Phase 2 완료 → Phase 3 착수: 답변 데이터가 있어야 답변 포함 포토카드와 공약 아카이브가 가능하다.
- Phase 3의 정렬 로직(F3-1)은 Phase 1 완료 후 Phase 2와 병행 개발 가능하다.

### 구현 순서

| 순서 | 항목 | 설명 |
|------|------|------|
| 1 | F1-1, F1-2, F1-3 | 유권자 핵심 루프 전체 |
| 2 | F2-1, F2-2, F2-3 | 후보 인증 + 대시보드 + 답변 |
| 3 | F2-4 | "약속합니다" 기능 |
| 4 | F3-1 | 정렬 로직 (Phase 2와 병행 가능) |
| 5 | F3-2 | 포토카드 |
| 6 | F3-3 | 공약 아카이브 |

### 선거 일정 기준 최소 런칭 범위

선거일까지 시간이 부족할 경우, **Phase 1 + Phase 2 (F2-1 ~ F2-3)**가 최소 바이럴 루프를 완성하는 최소 범위이다. 유권자가 글을 쓰고 고유 링크를 받고, 후보가 답변하고, 유권자가 알림을 받아 확인하는 핵심 순환이 돌아간다.

"약속합니다", 포토카드, 공약 아카이브는 선거 전 추가 또는 선거 후 업데이트로 대응 가능하다.

## 9. 향후 검토 사항

아래 항목은 현재 개발 범위에 포함하지 않으나, 서비스 확장 시 검토할 수 있다.

- **지도 뷰**: 동네별 목소리 밀도를 파동(ripple) 시각화로 표현. 리스트 뷰와 탭 전환.
- **누적 번호**: 포토카드에 "OO동에서 N번째 목소리" 표시. 초기 얼리어답터 자부심 + 후기 운동 규모감.
- **답변률 공개**: 후보별 답변률을 프로필에 표시. 경쟁 심리 유도.
- **선거 후 전환**: 당선자의 공약 이행 추적 플랫폼 또는 상시 주민 소통 채널로의 서비스 전환.
- **QR 코드 연동**: 유세 현장에서 QR 코드를 통한 서비스 접속 유도.
