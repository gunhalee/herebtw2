# 여기 근데 - 서비스 기획안

## 1. 서비스 개요

| 항목 | 내용 |
| --- | --- |
| 서비스명 | 여기 근데 |
| 형태 | PWA (Progressive Web App) |
| 한 줄 정의 | 위치 기반 지역 발언 플랫폼. 현장에서 남긴 짧은 목소리를 행정동명과 거리 기반으로 읽고 확산할 수 있는 서비스 |
| 핵심 가치 | 현장성, 간결함, 직접성, 소수의견 보호 |

**여기 근데**는 지역 현장에서 느낀 문제의식과 제안을 짧고 명확하게 남기고, 같은 생활권 안에서 읽히고 공감받도록 설계된 서비스다. 앱 설치 장벽 없이 브라우저에서 바로 접근할 수 있으며, 홈 화면 추가 시 앱처럼 사용할 수 있다.

## 2. 해결하려는 문제

- 기존 커뮤니티는 다수 의견과 자극적인 발언이 더 쉽게 확산된다.
- 지역 현장의 생활권 이슈는 온라인에 흩어져 있지만 가볍게 모이고 읽히는 구조가 부족하다.
- 현장과 무관한 발언이 과도하게 소비되며 실제 지역 의제는 묻히기 쉽다.

여기 근데는 이 문제를 해결하기 위해 발언의 길이를 줄이고, 위치 기반 인증을 적용하며, 반응 구조를 단순화한다. 이를 통해 "누가 더 크게 말했는가"보다 "누가 현장에서 무엇을 말했는가"에 집중하게 만든다.

## 3. 핵심 기능

### 3-1. 위치 기반 발언

- 현재 위치 GPS 인증 후에만 포스트 작성 가능
- GPS 좌표 원본은 저장하지 않고 행정동 단위 이름만 기록
- 포스트는 100자 이내 텍스트로 제한
- 사진 첨부는 제공하지 않음

이 구조는 현장성을 확보하면서도 개인정보 노출 위험을 줄이는 방향으로 설계한다.

### 3-2. 작성 직후 이미지 카드 다운로드

| 요소 | 설명 |
| --- | --- |
| 이미지 카드 다운로드 | 포스트 작성 직후 문장을 카드 이미지로 저장 가능 |
| 맞아요 | 위치 권한 허용 상태에서 가능. 익명 디바이스 아이디 기준 포스트당 1회 |
| 신고 | 위치 권한 허용 상태에서 가능. 익명 디바이스 아이디 기준 포스트당 1회 |
| 계정 | 회원가입 및 소셜 로그인 없이 이용 |

- 작성 직후 바로 저장할 수 있어 사용자가 자신의 발언을 보관하거나 외부 채널에 가져가기 쉽다.
- `맞아요`와 신고는 최소한의 상호작용만 남기고 운영 검수로 관리한다.
- 계정 절차를 제거해 서비스 진입 장벽을 낮춘다.
- 댓글을 제거해 발언 하나하나가 독립적으로 남도록 만든다.

### 3-3. 정치인 반응 기능

- 정치인 인증 계정은 `맞아요`와 `답글` 두 가지 반응 가능
- 정치인 맞아요는 파란 인증 뱃지와 함께 노출
- 정치인 답글은 포스트 하단에 고정 표시
- 답글에는 정치인 이름, 직책, 날짜를 함께 표기
- 정치인 답변이 달린 포스트는 별도 탭에서 조회 가능

정치인의 반응은 단순한 참여가 아니라 "읽었고 응답했다"는 신호로 작동해야 한다. 따라서 일반 반응보다 더 명확하고 공적으로 보이도록 설계한다.

### 3-4. 거리 기반 리스트 UI

- 메인 화면은 내 주변 포스트 거리순 리스트 구조
- 각 카드에는 행정동명과 거리 정보 노출
- 포스트 목록 기본 10개 노출
- 기본 정렬은 거리순
- 포스트를 탭하면 `여긴 근데` 상세 화면으로 전환
- `+` 버튼 선택 시 글쓰기 화면 진입

거리 기반 리스트 UI는 복잡한 지도나 구획 탐색 없이도, 사용자가 얼마나 가까운 곳의 발언인지 즉시 이해하도록 돕는다.

### 3-5. 민주적 설계 원칙

- **침묵이 올린다**: 반박 경쟁보다 독립 발언이 더 오래 보이도록 설계
- **거리가 목소리를 키운다**: 현장에 가까운 이용자일수록 더 강한 관련성을 가짐
- **소수의견을 보호한다**: 현장 내 유일한 발언은 낮은 완독 수여도 정치인 피드에 노출

여기 근데는 참여량보다 현장성과 다양성을 우선하는 구조를 지향한다.

## 4. 계정 시스템

### 4-1. 일반 사용자

- 별도 회원가입이나 소셜 로그인 없이 바로 사용
- 익명 디바이스 아이디로 최소 기능 상태를 유지
- 위치 권한 허용 시 포스트 작성 가능
- 위치 권한 허용 시 `맞아요`와 신고 가능
- 위치 권한 거부 시 읽기만 가능
- 모든 포스트는 익명으로 노출
- 사용자는 주변 포스트 읽기, 포스트 작성, `맞아요`, 신고, 이미지 카드 다운로드 가능

### 4-2. 계정 없는 사용 구조

- MVP 1차에서는 개인 프로필을 두지 않음
- 작성 후에는 포스트가 완전히 익명으로 섞이며 별도 `내 글` 식별 UI를 두지 않음
- 다만 작성 직후 3분 동안은 삭제 가능 상태를 제공
- 계정 기반 관리 기능은 후속 단계에서 검토

### 4-3. 정치인 계정

- 이메일로 가입 신청
- 제출 항목: 이름, 소속 기관, 선거구, 공식 연락처
- 운영자가 공공 정보와 대조 후 수동 승인
- 승인 완료 후 이메일 알림 발송
- 사용 가능 기능: 맞아요, 답글 작성, 공유 링크 복사, 이미지 저장

정치인 계정은 신뢰가 핵심이므로 자동 인증 대신 수동 검증 방식을 채택한다.

## 5. 이미지 카드 활용

MVP 1차에서는 외부 SNS 연동보다 작성 직후의 카드 다운로드 경험에 집중한다.

- 일반 사용자: 포스트 작성 직후 이미지 카드 다운로드
- 자동 포스팅은 제공하지 않으며 사용자가 직접 활용
- 초기 단계에서는 외부 연동보다 발언 생성 경험을 우선 검증

이 전략은 초기 운영 복잡도를 낮추면서도 최소한의 확산 가능성은 확보하는 방식이다.

## 6. 기술 스택

| 영역 | 기술 |
| --- | --- |
| 프론트엔드 | Next.js + Tailwind CSS |
| 지도 | 사용하지 않음 (거리 정보 기반 리스트 UI) |
| 백엔드/DB | Supabase + PostgreSQL + PostGIS |
| 인증 | 계정 체계 없음, 익명 디바이스 아이디 기반 제어 |
| 파일 저장 | Supabase Storage |
| 이미지 생성 | html2canvas 또는 Vercel OG |
| 배포 | Vercel |

기술 선택의 기준은 빠른 MVP 구축, 위치 기반 처리, 운영 단순성, 배포 자동화다.

## 7. 개발 로드맵

| 단계 | 기간 | 범위 |
| --- | --- | --- |
| 1단계 MVP | 1~2개월 | 거리 기반 리스트, 위치 인증 글쓰기, 익명 디바이스 아이디 기반 사용 구조, `맞아요`/신고, 작성 직후 이미지 카드 다운로드 |
| 2단계 정치인 기능 | 2~3개월 | 정치인 수동 가입, 인증 뱃지, 답글 기능 |
| 3단계 공유 강화 | 3~4개월 | 공유 링크, OG 카드 저장, PWA 홈 화면 추가 |
| 4단계 고도화 | 4개월 이후 | 의제 연결 알고리즘, 행정동 통계, 정치인 대시보드 |

단계별로 핵심 가설을 검증하면서 기능을 확장하는 방식이 적합하다.

## 8. 핵심 지표

| 지표 | 정의 | 의미 |
| --- | --- | --- |
| 작성 완료율 | 작성 화면 진입 후 실제 포스트 작성까지 이어진 비율 | 발화 경험의 마찰 정도 판단 |
| 주변 포스트 조회 전환율 | 메인 진입 후 주변 포스트 목록 확인까지 이어진 비율 | 홈 구조의 직관성 판단 |
| 재방문율 | 일주일 내 2회 이상 접속한 사용자 비율 | 습관화 여부 판단 |
| 카드 다운로드율 | 작성 완료 후 이미지 카드 저장까지 이어진 비율 | 외부 확산 가능성 판단 |
| 지역 발언 축적도 | 특정 생활권 내 일정 기간 누적된 포스트 수 | 지역 플랫폼 가능성 판단 |

## 9. 기대 효과

- 지역 현안이 현장 기반으로 더 빠르게 수집된다.
- 시민은 긴 글이나 논쟁 없이도 의견을 남기고 공감할 수 있다.
- 계정 없는 구조 덕분에 초기 참여 장벽을 낮출 수 있다.
- 반응과 신고를 최소 단위로 두면서도 운영 검수 체계를 유지할 수 있다.

여기 근데는 지역 사회의 짧지만 실제적인 발언을 더 쉽게 남기고 읽히게 만드는 경량 지역 발언 인터페이스를 목표로 한다.
