# 여기 근데 한마디 할게요 — 개발 로드맵

## 서비스 개요

지방선거 기초지방자치단체 의원 후보 출마자와 지역 주민 간의 소통 플랫폼.
유권자는 계정 없이 위치 기반 익명 글을 남기고, 인증된 후보만 답글을 달 수 있는 비대칭 소통 구조.

### 핵심 설계 원칙

- **유권자 경험은 단순하게 유지한다.** 복잡한 기능은 후보 쪽에만 얹는다.
- **유권자가 의제의 주인이다.** 좋아요 기반 정렬로 어떤 목소리가 중요한지 유권자 커뮤니티가 결정한다.
- **모든 기능은 "기록"으로 수렴한다.** 글, 답변, 약속은 지워지지 않으며, 기록 자체가 책임을 만든다.

### 현재 상태 (v0)

- 위치 권한 기반 자동 위치 입력
- 익명 글 작성 및 목록 표시
- 좋아요 기능

## Phase 1 — 유권자 경험 핵심 루프

> 목표: 유권자가 "내 글을 다시 찾고, 답변을 기다릴 수 있는" 최소한의 연결고리를 만든다.

### 1-1. 고유 링크 생성 및 제공

- 글 작성 완료 시 UUID 기반 고유 링크 자동 생성
- 완료 화면에서 링크 복사 버튼 제공
- URL 형태: `herebtw.vercel.app/v/{uuid}`

**구현 사항**
- 글 테이블에 UUID 필드 추가 (PK와 별도, 외부 노출용)
- 글 작성 완료 시 결과 화면 UI: 체크 아이콘 + "목소리가 전달되었습니다" + 링크 복사 버튼
- 클립보드 복사 후 "복사됨" 피드백

### 1-2. 고유 링크 페이지

- 해당 글의 내용, 위치, 작성 시간 표시
- 상태 표시: **전달됨** → **답변 완료** (2단계)
  - 전달됨: 글 작성 직후 기본 상태
  - 답변 완료: 후보가 답변을 등록한 상태 (답변 내용 함께 표시)
- 답변이 달린 경우 후보명 + 인증 배지 + 답변 본문 + "약속합니다" 배지(해당 시) 표시

**구현 사항**
- `/v/{uuid}` 라우트 추가
- 글 상태 필드: `status` enum (`delivered`, `replied`)
- 답변 등록 시 자동으로 `replied`로 전환

### 1-3. 이메일 알림

- 글 작성 시 이메일 입력 필드 (선택 사항)
- 안내 문구: "답변이 달리면 알려드릴까요?"
- 후보가 답변 등록 시, 이메일이 있는 글이면 알림 발송

**구현 사항**
- 글 테이블에 `notification_email` 필드 추가 (nullable)
- 이메일 발송 서비스 연동 (Resend 또는 동급 서비스)
- 발송 내용: "당신의 목소리에 답변이 달렸습니다" + 고유 링크
- 이메일은 알림 용도로만 사용, 별도 저장·마케팅 활용 없음 명시

### Phase 1 완료 기준

- [ ] 글 작성 → 고유 링크 생성 → 복사 가능
- [ ] 고유 링크 페이지에서 글 내용 + 상태 확인 가능
- [ ] 답변 등록 시 상태 전환 + 답변 표시
- [ ] 이메일 입력한 유권자에게 답변 알림 발송

## Phase 2 — 후보 인증 및 답변 시스템

> 목표: 후보가 인증하고, 주민 글에 답변하고, 공약을 남길 수 있는 구조를 만든다.

### 2-1. 후보 인증 플로우

- 관리자가 후보 계정 생성 (초대 코드 또는 직접 등록)
- 후보 로그인 (이메일 + 비밀번호 또는 OAuth)
- 인증 완료 시 "첫 마디" 작성 필수
  - 첫 마디는 주민 글과 동일한 형태로 표시
  - 글 목록에서 해당 지역구 최상단에 고정
  - 후보명 + 인증 배지 표시
  - 예시: `[김민수 후보 ✓] "능평동 주민 여러분, 여러분의 이야기를 듣겠습니다."`

**구현 사항**
- 후보 테이블: `id`, `name`, `district`, `auth_method`, `first_message`, `created_at`
- 인증/인가 시스템 (JWT 또는 세션 기반)
- 첫 마디 글은 일반 글 테이블에 저장하되 `is_pinned: true`, `author_type: candidate` 플래그
- 목록 렌더링 시 해당 지역구의 pinned 글을 최상단에 표시

### 2-2. 후보 대시보드

- 자기 지역구 글 목록 전용 뷰
- 기본 정렬: 좋아요 많은 미답변 글 최상단
- 각 글에 답변 여부 표시
- 미답변 + 좋아요 N개 이상 글 강조 (배경색 또는 뱃지)

**구현 사항**
- `/candidate/dashboard` 라우트 (인증 필요)
- 정렬 로직: 미답변 우선 → 좋아요순
- 글 카드에 답변 상태 아이콘 + 좋아요 수 강조 표시
- 간단한 통계 표시: 총 글 수, 답변 완료 수, 미답변 수

### 2-3. 답변 작성

- 후보 대시보드 또는 일반 글 목록에서 답변 작성
- **200자 제한**
- **수정 불가** — 작성 완료 시 확인 모달: "답변은 수정할 수 없습니다. 등록하시겠습니까?"
- 등록 즉시 해당 글 상태 `replied`로 전환
- 이메일 알림 트리거 (Phase 1-3 연동)

**구현 사항**
- 답변 테이블: `id`, `post_id` (FK), `candidate_id` (FK), `content` (max 200), `is_promise`, `promise_deadline`, `created_at`
- 답변 작성 UI: 텍스트 입력 + 잔여 글자 수 표시 + 확인 모달
- 글 상태 자동 전환 로직
- 이메일 발송 트리거

### 2-4. "약속합니다" 기능

- 답변 작성 시 "이 답변은 약속입니다" 토글
- 토글 활성화 시 기한 입력: 드롭다운으로 "당선 후 3개월 / 6개월 / 1년 / 직접 입력"
- 답변에 "약속합니다" 배지 표시
- 후보 프로필 페이지에 약속 목록 자동 축적

**구현 사항**
- 답변 테이블의 `is_promise` (boolean), `promise_deadline` (date, nullable) 필드 활용
- 답변 UI에 토글 + 기한 선택 추가
- 약속 배지 컴포넌트
- 후보 프로필 페이지: `/candidate/{id}` — 약속 목록 + 자동 요약 ("주민 N명에게 답하고 M개의 약속을 한 후보")

### Phase 2 완료 기준

- [ ] 후보 로그인 → 첫 마디 작성 → 글 목록 최상단 고정
- [ ] 후보 대시보드에서 미답변 글 강조 확인
- [ ] 답변 작성 (200자 제한, 수정 불가, 확인 모달)
- [ ] "약속합니다" 토글 + 기한 → 배지 표시
- [ ] 답변 등록 시 글 상태 전환 + 이메일 알림 발송
- [ ] 후보 프로필 페이지에서 약속 목록 확인

## Phase 3 — 정렬, 포토카드, 공약 아카이브

> 목표: 유권자 주도권을 구조적으로 보장하고, 양방향 바이럴 장치와 선거 이후 연속성을 만든다.

### 3-1. 거리 기반 정렬 + 동네 내 좋아요순

- 1단계: 접속 위치 기준 가까운 동네 글부터 표시
- 2단계: 동네 내에서 좋아요순 정렬 (기본) / 최신순 전환 가능
- 시간 감쇠: 오래된 좋아요의 가중치 점진적 감소
- 초기 대응: 동네 내 글 수가 임계치(예: 10개) 미만이면 최신순 기본
- 미답변 강조: 좋아요 N개 이상 + 미답변 글에 "주민 N명이 관심을 보인 목소리입니다" 배지

**구현 사항**
- 정렬 스코어 함수: `score = likes × decay(time) `
- 시간 감쇠 함수: 예) `decay = 1 / (1 + hours_since_post / 72)` (3일 반감기)
- 동네 클러스터링: 행정동 단위 그룹핑 (위치 데이터에서 동 단위 추출)
- 정렬 모드 전환 UI: "공감순 / 최신순" 토글
- 임계치 기반 기본 정렬 모드 분기 로직

### 3-2. 포토카드

- 상태별 다른 비주얼:
  - **전달됨**: 유권자 글만 표시. 톤: 차분한 기본 카드
  - **답변 완료**: 유권자 글 + 후보 답변. 톤: 대화가 완성된 느낌. "약속합니다" 배지 포함 (해당 시)
- 공통 요소: 위치, 서비스 URL/QR, 서비스 로고
- 다운로드 진입점:
  - 글 작성 직후 완료 화면
  - 고유 링크 페이지
  - 후보 대시보드 (후보용 카드)
- 유권자 카드 / 후보 카드 시점 분리:
  - 유권자 카드: "나는 목소리를 냈다"
  - 후보 카드: "나는 주민의 목소리에 답했다"

**구현 사항**
- 서버사이드 이미지 생성 (satori + @resvg/resvg-js 또는 Puppeteer)
- API 엔드포인트: `GET /api/card/{uuid}?type=voter|candidate` → PNG 반환
- OG 메타태그 동적 생성: 고유 링크 공유 시 SNS 미리보기에 카드 이미지 표시
- 카드 디자인 템플릿 2종 (전달됨, 답변완료) × 2시점 (유권자, 후보)
- 다운로드 버튼 컴포넌트 (공통)

### 3-3. 공약 아카이브 페이지

- 후보별 약속 모음 페이지
- 각 약속에 원본 주민 글 + 후보 답변 + 기한 표시
- 기한 기준 D+일수 자동 표시 (선거일 이후부터 카운트)
- 후보 프로필 자동 요약: "주민 N명에게 답하고 M개의 약속을 한 후보"
- 선거 이후에도 접근 가능 — 서비스의 연속성 장치

**구현 사항**
- `/candidate/{id}/promises` 라우트
- 약속 목록 쿼리: 답변 테이블에서 `is_promise = true` 필터
- D+일수 계산: 선거일(설정값) 기준 경과일 표시
- 요약 통계 컴포넌트: 총 답변 수, 약속 수, 약속 이행 기한 분포

### Phase 3 완료 기준

- [ ] 거리 기반 정렬 적용 — 가까운 동네 글부터 표시
- [ ] 동네 내 좋아요순/최신순 전환 동작
- [ ] 미답변 강조 배지 표시
- [ ] 포토카드 2종(전달됨/답변완료) 이미지 생성 및 다운로드
- [ ] OG 메타태그로 SNS 공유 시 카드 미리보기 표시
- [ ] 공약 아카이브 페이지에서 약속 목록 + D+일수 확인

## 일정 및 우선순위

### 의존 관계

```
Phase 1 ──→ Phase 2 ──→ Phase 3
(고유 링크)   (답변 시스템)  (포토카드·아카이브)
                ↓
          이메일 알림 연동
```

Phase 1이 완료되어야 Phase 2의 답변 알림이 의미를 가지며, Phase 2가 완료되어야 Phase 3의 답변 포함 포토카드와 공약 아카이브가 가능하다.

### 권장 구현 순서

| 순서 | 항목 | 비고 |
|------|------|------|
| 1 | Phase 1 전체 | 유권자 핵심 루프 확보 |
| 2 | Phase 2-1, 2-2, 2-3 | 후보 인증 + 대시보드 + 답변 |
| 3 | Phase 2-4 | "약속합니다" 기능 |
| 4 | Phase 3-1 | 정렬 로직 |
| 5 | Phase 3-2 | 포토카드 |
| 6 | Phase 3-3 | 공약 아카이브 |

### 선거 일정 기준 최소 런칭 범위

선거일까지 시간이 부족할 경우, Phase 1 + Phase 2(2-1~2-3)가 **최소 바이럴 루프**를 만드는 최소 범위.
"약속합니다", 포토카드, 공약 아카이브는 선거 전 추가 또는 선거 후 업데이트로 대응 가능.
