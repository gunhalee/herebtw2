# 여기 근데 모바일 QA 체크리스트

## 1. 기본 실행

- `npm install`
- `npm run dev`
- 브라우저에서 `http://localhost:3000` 접속
- 개발자도구 모바일 뷰 활성화
- 권장 1차 확인 폭: `390 x 844`, `375 x 812`, `360 x 800`

## 2. 홈 화면

- 첫 화면이 좌우로 흔들리지 않고 한 열 레이아웃으로 보이는지 확인
- 상단 배너, breadcrumb, 지역 카드, 목록 시트가 한 화면 리듬으로 이어지는지 확인
- 지역 카드 텍스트가 줄바꿈되더라도 카드 높이가 과하게 깨지지 않는지 확인
- 카드 버튼이 손가락으로 누르기 충분한 높이와 간격을 가지는지 확인
- breadcrumb가 좁은 폭에서도 두 줄 이내로 자연스럽게 감기는지 확인

## 3. 지역 카드

- 카드가 1열로 보이는지 확인
- 선택된 카드는 배경/테두리 차이로 바로 구분되는지 확인
- `탐색 가능한 지역`, `현재 선택 지역`, `글 N` 정보가 한눈에 읽히는지 확인
- 카드 내부 텍스트가 버튼 경계 밖으로 넘치지 않는지 확인

## 4. 글 목록 시트

- 시트 제목, 설명, 카드 목록 간 간격이 일정한지 확인
- 글 카드 본문 100자가 작은 화면에서도 읽기 가능한지 확인
- 메타 정보(`행정동명`, `상대시간`, `맞아요 수`)가 한 줄에 안 들어가도 자연스럽게 감기는지 확인
- 빈 상태, 로딩 상태, 에러 상태 카드 폭과 간격이 목록 카드와 어울리는지 확인

## 5. 포스트 상세 시트

- 상세 본문 카드와 액션 버튼 영역이 세로로 안정적으로 쌓이는지 확인
- `맞아요`, `신고`, `삭제 mm:ss` 버튼이 모두 풀폭으로 보이는지 확인
- 버튼 간 간격이 너무 좁거나 너무 넓지 않은지 확인
- 삭제 카운트다운 텍스트가 버튼 안에서 잘리지 않는지 확인

## 6. 작성 화면

- 상단 소개 카드, 작성 폼, 성공 액션 카드가 같은 폭으로 정렬되는지 확인
- textarea가 모바일에서 너무 낮지 않고, 키보드 노출 시에도 사용 가능한지 확인
- `등록하기` 버튼이 풀폭으로 노출되는지 확인
- `이미지 카드 다운로드`, `삭제` 액션이 세로 스택으로 안정적으로 보이는지 확인
- 디바이스 ID 문구가 너무 길게 보여서 레이아웃을 깨지 않는지 확인

## 7. 최종 체크

- 모든 카드 radius, padding, 텍스트 크기가 일관적인지 확인
- 가로 스크롤이 생기지 않는지 확인
- 버튼 텍스트가 줄바꿈 없이 유지되는지 확인
- iPhone 폭과 Android 폭 모두에서 시각적 밀도가 과하지 않은지 확인
